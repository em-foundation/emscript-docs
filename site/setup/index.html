    Getting started with EM•Script

    EM•Script requires a cross-development environment comprising hosted  software tools will compile and load executable programs onto target  MCU hardware.  For the host, you'll use a PC running Windows, Linux, or MacOS; for the target, you can choose any MCU board for which a corresponding em$distro support package already exists.

    Before turning to the EM•Script SDK (described next), you should first install or upgrade the following tooling environments on your host PC:
    Node.js version 18.0.0 or later execute node --version to verify
    VS Code version 1.90.0 or later execute code --version to verify

    If you don't already have a recent version of the Git Bash shell, you should also install Git for Windows.  To verify your setup, ensure that the node and code commands from the previous table operate correctly under Git Bash.

    Do not proceed forward if these verification checks should fail !!!

    Software development kit

    You can provision the latest EM•Script SDK onto your host PC in three simple steps:

      download a workspace pre-populated with packages of EM•Script target-content
      install EM•Script host-tooling along with some 3rd party compilers and loaders
      verify that you can build (and optionally run) a sample EM•Script program

    Step relies upon the git command to initially populate (and subsequently update) a special emscript-sdk folder on your host PC:

    [~]
$ git clone https://github.com/em-foundation/emscript-sdk.git
$ cd emscript-sdk
[~/emscript-sdk]
$ ▮

    In general, you can maintain multiple emscript-sdk installations located anywhere on your PC – though we recommend starting with your $HOME folder [~] as illustrated above.  Once inside ~/emscript-sdk, you can use other git sub-commands to upgrade your EM•Script installation [git pull] as well as switch to particular tagged releases [git checkout v26.1.0].

    Step uses the Node Package Manager npm command to install tooling artifacts into your current emscript-sdk folder:

    [~/emscript-sdk]
$ npm clean-install

    Initial execution of this command will create the special node_modules sub-folder used by Node.js as well as a tools sub-folder used by EM•Script.  Use the npm list command to show the current set of installed packages, which should track the dependencies called out within the top-level emscript-sdk/package.json file.

    Step finally verifies your EM•Script installation by building a small program: 

    [~/emscript-sdk]
$ npm run verify-build

    Target MCU hardware

    The SDK contains all tooling needed to compile and load EM•Script programs targeting different MCU development boards.  While we encourage you to purchase one of these boards, you can still learn about EM•Script by building (but not running) some sample programs.

    The Texas Instruments CC2340R5 wireless MCU features an Arm Cortex-M0+ CPU together with a familiar suite of peripherals – including a generic 2.4 GHz radio with BLE 5.x support. Texas Instruments also offers an inexpensive LP-EM-CC2340R5 evaluation board in their familiar LaunchPad format – available from TI as well as their distributors.

    You should also purchase this emulator board from TI – unless you already own a "classic" TI LaunchPad with on-board XDS110 support. In that case, you can easily connect a legacy LP to your new LP-EM-CC2340R5 board using a cable supplied by TI.  If you haven't used an XDS110 before, run the emscript-sdk/tools/ti-uniflash/one_time_setup script.

    We'll soon verify that your LP-EM-CC2340R5 hardware works in concert with the SDK by building and loading a sample EM•Script program.

    VS Code extension

    To flatten your learning curve, we strongly encourage use of our EM•Script Extension, which you can install from the command-line prior to launching VS Code:

    [~/emscript-sdk]
$ code --install-extension emscript*.vsix
    Installing extensions...
Extension â€˜emscript-26.0.1.202502091752.vsixâ€™ was successfully installed.
    
[~/emscript-sdk]
$ code .

    If all goes well, you should see a top-level listing of the ~/emscript-sdk folder together with a short notification that the EM•Script Extension has activated.

    You can learn more by navigating to the EXTENSIONS > INSTALLED side-panel or else by visiting the EM•Script page at the VS Code Marketplace.

    From here, you should drill-down into the ~/emscript-sdk/workspace sub-folder and explore the software content which EM•Script can build and load onto your target MCU hardware.

    Command line

    The SDK contains an emscript command-line tool which compiles and loads programs for your target MCU hardware.  Used internally by npm run verify-build as part of installation step , you can access the emscript command directly from your PC's shell.

    You'll find the emscript executable at emscript-sdk/node_modules/.bin/emscript;  for convenience, amend your shell's $PATH or else reference emscript with a named link or alias:

    $ alias emscript='~/emscript-sdk/node_modules/.bin/emscript'
$ emscript --version
26.0.1.202502071929

    The emscript build sub-command will serve as your primary entry-point when working with EM•Script.  Typically invoked inside the emscript-sdk/workspace sub-folder, let's now build the same  target program used to verify SDK installation:
